LoRA Architecture & Composition
Adapter Vault

LoRA adapters are the system's institutional memory at the model level. Where Qdrant stores what happened, LoRAs encode how the universe speaks — the specific narrative voice, vocabulary, and stylistic fingerprint of the fiction.

The Three-Tier Hierarchy

T1
UNIVERSE
universe_voice — r=64 · MERGED · Always active
Trained on all universe-level world-building documents. The foundational voice of the entire Dark Matter Universe.
Every other adapter stacks on top of this. Because it is merged into the base model weights at deployment time, it adds zero VRAM overhead during inference. All agents inherit this voice.
T2
SERIES
darkmatter_s01 · darkmatter_s02 — r=32 · weight: 1.0
Series-specific lore, recurring locations, faction dynamics, season arc knowledge. Swapped per generation job.
Only one T2 LLM adapter should be active at a time. Applying both series adapters simultaneously causes interference and degrades output coherence.
T3
CHARACTER
kael_theron_voice · sera_vance_voice — r=16 · weight: 0.7 max
Individual character voice adapters. Applied at reduced weight to preserve series tone.
Never combine two T3 LLM adapters in a single stack — interference risk. Stack T1 (merged) + T2 (1.0) + one T3 (≤0.7) maximum per agent run.

Agent Orchestration Reference
The 6-Agent Pipeline

The pipeline is a LangGraph directed graph. The Showrunner is the supervisor node — it receives the brief, delegates to writers, collects output, and coordinates the editorial pass. Each agent has its own model, LoRA stack, and memory configuration.

Agent Roster

SHOWRUNNER
Qwen2.5-32B · High capability orchestrator
universe_voice

Orchestrator and supervisor. Receives the episode brief, breaks it into a beat sheet, assigns scenes to the Head Writer, manages continuity at episode level, and coordinates the editorial pass.

WORLD BUILDER
Qwen2.5-14B · Long context
universe_voicedarkmatter_s02

Expands series bible entries, creates new locations and faction lore on demand. Invoked when the Showrunner requires new canonical world-building not present in existing lore.

HEAD WRITER
Qwen2.5-14B · Structured output
universe_voicedarkmatter_s02

Produces beat sheets and scene outlines from the Showrunner's episode brief. Specifies story function, primary characters, setting, and character state (in/out) for each scene.

SCENE WRITER
Qwen2.5-14B · Creative generation
darkmatter_s02kael_theron_voice

Generates full scene scripts. RAG-queries Qdrant before each scene (7 chunks average). Receives Neo4j constraint feedback inline. LoRA stack swaps per character focus.

STORY EDITOR
Qwen2.5-7B · Fast evaluator
universe_voice

Reviews the full episode draft. Checks: character consistency, continuity violations, pacing. Returns pass/fail with specific inline notes. Triggers rewrites for failed scenes.

RESEARCHER
Qwen2.5-7B · RAG specialist
No LoRA — pure retrieval

Lore retrieval specialist. Queries Qdrant and Neo4j on behalf of other agents. Formats and ranks results. Does not generate narrative content — only retrieves and structures existing lore.

Production Walkthrough
Running a Generation Job

This walkthrough generates a complete episode of Dark Matter — Series 02, Episode 04. Each step shows what happens at the system level during a production run.

6-Stage Generation Walkthrough

STEP 01 — BRIEF SUBMISSION

Navigate to Generation → New Job. Select: Series: Dark Matter, Season: 02, Episode: 04.

Enter the one-line brief:

"S02E04 — Kael confronts Maren in the Vault antechamber. The Conclave's secret is revealed."
STEP 02 — SHOWRUNNER

The Showrunner receives the brief and produces a 12-scene beat sheet. Each beat specifies: story function, primary characters, setting, and character state going in/out.

Output: 12 beats · ~800 tokens · 8.2s · Qwen2.5-32B + universe_voice

STEP 03 — HEAD WRITER

The Head Writer expands each beat into a full scene outline — setting, action, dialogue purpose, character emotional state in/out. Serves as the blueprint for the Scene Writer.

Output: 12 outlines · ~3,200 tokens · 22s

STEP 04 — SCENE WRITER (×12)

The Scene Writer generates each scene in sequence. Before writing each scene, it executes:

1. RAG query to Qdrant (avg 7 chunks, 12ms) — retrieves relevant lore context
2. Neo4j constraint check — validates all referenced entities are alive/accessible
3. Generates scene with LoRA stack: darkmatter_s02 (1.0) + kael_theron_voice (0.7)

Output: 12 full scenes · ~28,000 tokens · ~8 min total

STEP 05 — STORY EDITOR

The Story Editor reviews the complete draft. It checks character consistency, continuity violations, and pacing. Issues inline correction notes. Scenes that fail are automatically requeued for rewrite with the constraint injected into the prompt.

Output: Pass 94% · 2 constraint flags · Revision notes · Qwen2.5-7B

STEP 06 — VISUAL PIPELINE

ComfyUI renders key frames for flagged and pivotal scenes using the FLUX LoRA stack. WAN2.2 generates motion clips for specified scenes if video is requested.

Output: 24 frames · 6 motion clips · darkmatter_style (0.55) + kael_identity (0.85)
